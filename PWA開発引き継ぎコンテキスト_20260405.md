### 🔴 優先度：高

#### history.jsのgetHistory表示不具合
- **症状**：履歴画面でデータが表示されない
- **原因（特定済み）**：GASのA列がDate型で返る場合、月フィルタの `indexOf(monthParam)` が一致しない
- **PWA側の対処**：`monthParam` はハイフン区切り（`YYYY-MM`）に統一済み、日付パースも両対応済み
- **残作業**：GAS側の `getHistory` 関数に `toDateStr` 変換を実装する必要あり
  ```javascript
  function toDateStr(value) {
    if (value instanceof Date) {
      return Utilities.formatDate(value, Session.getScriptTimeZone(), 'yyyy-MM-dd');
    }
    return String(value).replace(/\//g, '-');
  }
  ```
- **確認方法**：スマホのChromeで `chrome://inspect` → コンソールで `[history] GASレスポンス:` を確認

## 6. 技術メモ

### GAS通信の注意点

- **通信方式はGET**（`app.js:19`）。POSTが必要な場合は `app.js` の `callGAS` 関数を修正すること
- GASのdateフィールドはシート設定によって `Date型` / `YYYY/MM/DD文字列` / `YYYY-MM-DD文字列` のいずれかで返ってくる
- PWA側では `split(/[-\/]/)` で両対応済み

### localStorage キー一覧

| キー | 内容 | 使用箇所 |
|-----|------|---------|
| `uz_service_master` | サービスマスタ | sales.js |
| `uz_staff_master` | スタッフマスタ | clockin.js / sales.js |
| `uz_attendance_date` | 勤怠記録の日付 | clockin.js |
| `uz_attendance_data` | 勤怠記録データ | clockin.js |

### 日付フォーマット統一方針

- **PWA → GAS送信**：`YYYY-MM-DD`（ハイフン区切り）
- **GAS → PWA受信**：`YYYY/MM/DD` または `YYYY-MM-DD` の両方を受け入れる
- **画面表示**：`YYYY/MM/DD` に統一（`formatDate` 関数）

### 個別管理の送信仕様

- 売上本体：`addSales` 1件（合計金額）
- 個別行：`addSales` を `Promise.all` で並列送信
  - `serviceCode` / `serviceName`：売上本体と同じ
  - `memo`：`顧客名　メモテキスト`（全角スペース区切り）
  - `miscItemName`：空文字
- **0円モード**（金額=0 かつアコーディオン開）：本体を送らず個別行のみ送信
