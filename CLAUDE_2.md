## プロジェクト概要

**商品名**：ウルトラ財務くん LEO版（スマホアプリ版）
**形式**：PWA（Progressive Web App）
**対象ユーザー**：スナック・BAR等のITリテラシーが低い個人事業主
**コンセプト**：毎日ボタンを押すだけで確定申告に使える損益が自動で出る

**技術構成**：
```
フロントエンド：HTML / CSS / JavaScript（PWA）
バックエンド  ：Googleスプレッドシート
連携         ：Google Apps Script（GAS）経由でPOST/GET
```

## CSS設計ルール（必ず守ること）

### CSS変数で配色を一元管理する

`css/base.css` に以下の変数を定義し、全HTMLから参照すること。
配色の変更は `css/theme-default.css` の1ファイルのみで完結させる。
コード内にカラーコードを直書き禁止。

```css
/* css/base.css に定義する変数（例） */

  --color-primary: #1D9E75;       /* メインカラー（テーマで上書き） */
  --color-primary-light: #E1F5EE; /* 薄いメインカラー */
  --color-primary-dark: #0F6E56;  /* 濃いメインカラー */
  --color-surface: #ffffff;
  --color-background: #f5f5f5;
  --color-text-main: #1a1a1a;
  --color-text-sub: #6b7280;
  --color-border: #e5e7eb;
  --color-sales: #0F6E56;         /* 売上の色（プラス） */
  --color-cost: #993C1D;          /* コストの色（マイナス） */
  --color-profit-bg: #E1F5EE;     /* 経常利益の背景 */

  --radius-sm: 8px;
  --radius-md: 12px;
```

### テーマ切り替え設計
将来のナイト系（黒×ゴールド）への差し替えは `theme-dark.css` を1枚追加するだけで完結させる。

## スマホUI設計ルール

- **スマホ専用設計**（PCレイアウトは不要）
- `max-width: 430px; margin: 0 auto;` でスマホ幅に固定
- タップターゲットは最低44px以上
- フォントサイズは最小14px（スマホキーボード起動に影響するため）
- ボトムナビゲーション（4タブ固定）：ホーム・入力・損益・未収
- 入力中はボトムナビを `position: fixed; bottom: 0;` で固定
- ボタンは `height: 56px` の大きいサイズを基本とする

## 画面仕様

### ホーム画面（index.html）

**表示要素（上から順に）：**
1. ヘッダー：店名 + アプリ名
2. 月次損益カード：当月の5行サマリー（経常利益をハイライト）
3. アクションボタン4つ：売上を入れる・コストを入れる・損益を見る・未収買掛け
4. 最近の入力：直近5件（科目・日付・メモ・金額）
5. ボトムナビゲーション

**ダミーデータで起動すること（GAS接続前）**

---

### 売上入力画面（input-sales.html）

**入力フロー（1画面1〜3項目の原則）：**
1. 発生日（今日の日付を自動セット・変更時のみカレンダー表示）
2. サービス選択（マスタ登録済み3種をラジオボタン）
3. 金額入力（数字キーボードで直接入力）
4. 税率表示（「税率：10% ›」表示のみ・例外時のみ変更）
5. 決済日（マスタのルールで自動セット・「発生日と同じ」バッジ表示）
6. メモ（任意・過去入力をサジェスト）
7. 「入力する」ボタン（高さ56px）

**禁止事項：**
- 科目の自由入力（マスタ登録ベースのみ）
- 複数科目を1画面で同時入力

---

### コスト入力画面（input-cost.html）

**入力フロー：**
1. 発生日（今日の日付を自動セット）
2. 区分選択：「仕入原価」か「販管費」をラジオボタン
3. 科目選択：区分選択後に対応する科目一覧が展開
4. 金額入力
5. 税率表示（科目ごとにマスタ設定）
6. 決済日（マスタのルールで自動セット）
7. メモ（任意）
8. 「入力する」ボタン

---

### 月次損益サマリー画面（report.html）

**表示構造（5行固定）：**
```
売上         ¥XXX,XXX  [タップで内訳展開]
仕入原価     ¥XXX,XXX  [タップで内訳展開]
粗利         ¥XXX,XXX  （展開なし・結果行）
販管費       ¥XXX,XXX  [タップで内訳展開]
経常利益     ¥XXX,XXX  （展開なし・結果行・ハイライト）
```

**機能：**
- 月選択（プルダウン・確定済み月のみ）
- 「前年同月」比較ボタン（押下時のみ差分を色付きで表示）
- 年度累計タブ（同じ5行構造）

---

### 未収・買掛け一覧画面（unpaid.html）

**表示：**
- 未収フラグONの売上データを自動抽出
- 未払フラグONのコストデータを自動抽出
- 表示項目：区分・科目・金額・発生日・メモ
- 消込ボタン（タップで入金/支払い完了に更新）

---

## データ設計（GAS / スプレッドシート）

### シート構成（Googleスプレッドシート）
```
マスタシート    ：科目・サービス名・税率・決済日ルール
売上シート      ：発生日・科目・金額・税率・決済日・入金フラグ・メモ
コストシート    ：発生日・区分・科目・金額・税率・決済日・支払フラグ・メモ
集計シート      ：関数で損益を自動計算（手動編集禁止）
```

### GAS エンドポイント（設計）
```
POST /exec?action=addSales    → 売上シートに1行追記
POST /exec?action=addCost     → コストシートに1行追記
GET  /exec?action=getSummary&month=YYYY-MM  → 月次損益を返す
GET  /exec?action=getUnpaid   → 未収・未払一覧を返す
POST /exec?action=clearUnpaid → 消込フラグを更新
```

## 絶対にやってはいけないこと

- Excelのセル・数式をUI上に一切見せない
- VBA・マクロの使用（Coworkで代替するため）
- マスタ未登録の科目を自由入力で受け付ける
- 前月以前のデータをアプリ内で修正・削除できる仕様にする
- AIが自動で確定する機能（AI提案→ユーザー確認→確定の3ステップ必須）
- カラーコードをCSSファイル内に直書きする（変数経由のみ）
- `position: fixed` のモーダル（レイアウト崩れの原因）
